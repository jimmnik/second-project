# Настройка репозитория

1. Сделать папку репозиторием — git init
2. «Разгитить» папку, если что-то пошло не так, для этого нужно удалить скрытую подпапку .git. — rm -rf
* ключ -r (от англ. recursive — «рекурсивно») позволяет удалять папки вместе с их содержимым;
* ключ -f (от англ. force — «заставить») избавит вас от вопросов вроде «Вы точно хотите удалить этот файл? А этот? И этот тоже?».

>**Будьте осторожны: в подпапке .git хранится история изменений. Если удалить .git, то вся история проекта будет стёрта без возможности восстановления — останется только последняя версия файлов.**

3. Привязать удалённый репозиторий к локальному — git remote add origin git@github.com:%ИМЯ_АККАУНТА%/first-project.git 
Убедиться, что репозитории связаны, — git remote -v


# Работа с файлами

1. Проверить состояние репозитория — git status

2. Подготовить файлы к сохранению — git add
* Команда git add --all подготовит к сохранению сразу все файлы.

# Работа с коммитами

1. Выполнить коммит — git commit -m ‘Мой первый коммит!’

2. Просмотреть историю коммитов — git log

# Отправить изменения на удалённый репозиторий

1. Отправить изменения на удалённый репозиторий — git push

>В первый раз эту команду нужно вызвать с флагом -u и параметрами origin (имя удалённого репозитория) и main или master (название текущей ветки). Флаг -u свяжет локальную ветку с одноимённой удалённой. Как вы связывали локальный и удалённый репозитории в предыдущем уроке, так же и здесь нужно дополнительно связать ветки.

* git push -u origin main
